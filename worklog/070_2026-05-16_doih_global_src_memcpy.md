**Gotchas / Lessons:**

1. **Bead acceptance fixture is unreachable by MVP — but both proposers caught this.** The bead text said "t5_tr2_hashmap.ll line 153 compiles end-to-end", but `build/t5_tr2_hashmap.ll:15` shows the global is `<{ ptr, [24 x i8] }>` — a `ConstantStruct` with a leading `ptr` field, NOT a `ConstantDataArray`. `_extract_const_globals` (`module_walk.jl:269`) filters at `init isa LLVM.ConstantDataArray || continue`, silently skipping it. Even if struct-extraction were added, the leading `ptr` field has no Bennett wire semantics. Both Plan-Opus proposers independently flagged this as a BLOCKER (proposer A: claim 3, lines 16-25; proposer B: claim 3, lines 16-25 + lines 159 BLOCKER section). Honest scope tightening per the **Bennett-ixiz lesson** (worklog/069): ship narrow MVP that handles `[N x iM]` ArrayType globals, file a follow-up bead (`Bennett-zxhg` = "Bennett-doih-struct" descriptive name) for ConstantStruct extension. This is the same pattern as Bennett-37mt (Phase 1 scoping) and Bennett-ixiz (5-site scoping). **Take-home:** when the bead text and the source disagree, scope to what the source supports + file a precise follow-up. Don't pretend the literal acceptance fixture works.

2. **Threading `globals` as kwarg through `_convert_instruction` avoids the q04a contract trip.** `test_q04a_convert_instruction_contract.jl:58-61` pins `_convert_instruction`'s signature as the positional Tuple `{Instruction, Dict{_LLVMRef,Symbol}, Ref{Int}, Dict{_LLVMRef, Vector{IROperand}}}`. Adding `globals` positionally would have flipped the test. Adding as kwarg with empty-dict default preserves the contract exactly — `code_warntype` ignores kwargs in its positional-Tuple inspection. Verified empirically (9/9 q04a tests still green post-edit, no Tuple update needed). This was Proposer A's threading recommendation; took it verbatim per the synthesis. **Take-home:** when a positional contract is pinned by a test, default-valued kwargs are a clean way to extend without breaking it.

3. **Const-GEP src needs dual-path detection.** The dispatch site `if _src_reaches_global(src_v)` initially only checked `LLVMIsAGlobalVariable` and `ConstantExpr<GEP>`. The var-GEP reject fixture (`doih_global_src_var_gep_reject.ll`) uses a runtime GEP `getelementptr i8, ptr @gvg, i32 %xext` — that's an `Instruction`-typed GEP, NOT a ConstantExpr. Without an Instruction-GEP arm in `_src_reaches_global`, the case fell through to predicate 6 and failed with the generic "not alloca-backed" 37mt message instead of the precise `Bennett-doih-vargep` breadcrumb. Caught by the test failing with the wrong breadcrumb on first run (2/77 fails). Added a 3rd arm to `_src_reaches_global` to also recurse through `Instruction`-typed GEPs whose base reaches a global; once the dispatch routes to the doih arm, `_global_root_and_offset` returns `nothing` for the variable index and G5 emits the correct breadcrumb. **Take-home:** "reaches a global" is a structural predicate that must handle both compile-time (ConstantExpr) and runtime (Instruction) GEP wrappers; otherwise the dispatch fails open and the user sees a misleading error.

4. **Reused `_alloca_is_fresh` despite its name suggesting "memset-only".** The helper's first arg is typed `LLVM.Instruction` (generic), and the body is freshness-of-alloca semantics — agnostic to whether the gating call is memset or memcpy. Reusing it (G9) closes the 37mt-inherited hazard (37mt has no fresh-dst gate and would XOR-overlay onto pre-existing dst bytes if the user wrote to dst before the memcpy). doih is the first memcpy variant to apply this gate. The 4-arg name `memset_inst` in the helper's parameter is misleading post-doih; left unchanged because (a) the type is generic, (b) renaming would touch 9nwt's call site, (c) the docstring would need rewriting. Drive-by candidate for a future tidy bead but not load-bearing.

5. **No pre-existing reject tests flipped.** Ran `git grep -nE '@test_throws.*global|@test_throws.*8bys|Global-pointer src memcpy|global-variable pointer' test/` — empty result. No tests anywhere asserted the pre-doih global-rejection contract that needed flipping. Contrasts with the **Bennett-ixiz pattern** (4 reject-test flips); doih is cleaner because the pre-doih message was an `8bys` catch-all that no test specifically pinned. All 8 peer regressions (37mt 86/86, 9nwt 87/87, ixiz 53/53, munq 69/69, lqif 12/12, q04a 9/9, 8kno 8/8, gate-count 39/39) green.
