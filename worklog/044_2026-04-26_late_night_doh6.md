**Gotchas / Lessons:**

- **3+1 protocol catches stale doc numbers up-front.** Both the bead and the perf review quoted "x+0 → 98 gates" as the pre-fix baseline; live measurement showed 86 (fold=false) or 26 (fold=true). The bead was written 2026-04-22; subsequent baseline shifts (U27 ripple-default, U28 fold-default) moved the numbers without updating the bead. Pattern: **always live-measure** before writing tests against the bead's claims. The tests pin the LIVE numbers, not the bead's.

- **Peephole-eliminated cases break tests that pin a specific lowering shape.** `test_epwy_fold_constants_default.jl:25` was originally pinning `x*Int8(1)` as the "fold helps 4×" example. After my peephole, both fold-on and fold-off collapse to the same 8-CNOT copy-out → fold delta disappears → test fails. Retargeting to `x*Int8(3)` (3.24× ratio, partial-product Toffolis where fold still wins) preserves the test's intent. **Pattern**: when adding a peephole, grep for tests that pin the pre-peephole behaviour as a "feature" and retarget them.

- **Oracle off-by-one for two's-complement: `Int8(~UInt8(...))` fails for values ≥ 128.** First version of `test_5qrn`'s `x ⊻ -1` oracle was `Int8(~UInt8(reinterpret(UInt8, n)))` — `Int8(uint8_val)` constructor checks range and throws when `uint8_val ≥ 128`. Failed 257 of 256 inputs (each side of zero). Fix: `reinterpret(Int8, ~reinterpret(UInt8, n))`. Pattern: any test oracle that bit-fiddles signed/unsigned types should `reinterpret`, not construct.

- **`simulate(c, a, b)` doesn't exist — use `simulate(c, (a, b))`.** Multi-arg circuits take a Tuple. Easy to forget. The error message from MethodError shows the available signatures inline, so failure is fast.

- **The peephole helpers cluster nicely as one block before `lower_binop!`.** `_wmask`, `_const_value_mod`, `_emit_copy_out!`, `_identity_emit_for_const`, `_try_identity_peephole!` — five small functions, ~70 LOC including the prologue comment that explains the soft-float NaN safety reasoning. Grouped together so a future reader sees the rationale + machinery in one place.

- **`Int64(-1) & mask` overflow safety**: Used `reinterpret(UInt64, Int64(op.value))` to get the bit pattern of constants without sign-extension surprises. At W=64 with allones constant `-1`, this gives `0xFFFFFFFFFFFFFFFF` which masked with `typemax(UInt64)` is itself. Verified by the W=64 test cases (`x & Int64(-1)`, etc.).

**Rejected alternatives:**

The leaf adders are also called from `lower_mul_wide!` (multiplier's accumulator add) and Karatsuba — those callers pass data-dependent wire vectors with no const provenance. Without `warnonly`, every unrendered export would error the build. `checkdocs = :none` similarly disables the doc-coverage check. These are the conservative defaults for "partial Documenter coverage that we'll grow over time" — opposite of the strict-mode defaults Documenter ships with.
