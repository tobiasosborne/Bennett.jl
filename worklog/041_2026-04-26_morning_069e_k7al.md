**Gotchas / Lessons:**

- **Width=0 IS a real sentinel for pointer-typed SSAs.** First version of k7al asserted `width >= 1` universally. test_memory_corpus L7c (pointer-typed phi) and L7d (pointer-select) errored on the first Pkg.test because `ir_extract.jl:1268` and `:1283` deliberately emit `width=0` for pointer-typed selects/phis (Bennett-cc0 M2b — pointers don't materialise as wires; routing lives in `ptr_provenance`). Relaxed IRSelect / IRPhi to `width >= 0` with a comment documenting the sentinel. *Generalisable lesson*: whenever adding "field >= 1" asserts, grep for `width = 0`, `w = 0`, `width=0` first to find any pre-existing sentinels. The grep would have caught both M2b sites pre-emptively.

- **Background `julia ... 2>&1 | tail -N` truncates `Pkg.test` output to a stack-trace fragment.** First run of k7al's full suite with `tail -8` showed only a stack-trace tail (Pkg.Operations frames) — exit code 0 and no obvious failure markers. Re-ran with `tail -100` and the actual `IRSelect: width=0 must be >= 1 (dest=p)` error surfaced. **Pattern to use**: when capturing background `Pkg.test`, use `grep -E "tests passed|tests failed|Test Failed|Got exception|Error During|^ERROR"` rather than line-count tail; OR use a larger window (≥150 lines). The line-count tail risks landing entirely inside a stack-trace and giving you no signal.

- **Inner constructors REPLACE the auto-generated default.** Once you add a single `function ...; new(...) end` inside a struct, the auto-generated all-fields default disappears. So the inner constructor must accept exactly the same positional signature as the field list, in field order. Easy to get right but easy to forget — a dropped field name in the constructor signature is silent.

- **`@test_throws "substring"` matches by message substring.** Used heavily in test_k7al — every rejection path tests both type AND a unique substring of the error message ("op=:bogus", "width=0", "ret_width=0", "incoming is empty", etc.). A future agent who edits an error message in ir_types.jl will see the substring assert fire as a localised diagnostic; the constructor body still does what it should but the test surface tells you exactly which message moved. Same convention used by Bennett-0zsk (chunk 039).

- **The validation tables ARE the documentation.** `_IR_BINOP_OPS = (:add, :sub, :mul, :and, :or, :xor, :shl, :lshr, :ashr, :udiv, :sdiv, :urem, :srem)` replaces the previously-stale comment `# :add, :sub, :mul, :and, :or, :xor, :shl, :lshr, :ashr` (which omitted the four div/rem variants). The new const can't go stale — adding a new op requires updating it, OR the constructor will error and the test will fail. Future agents adding IR ops: append to the matching const tuple AND update test_k7al's per-op happy-path testset.

**Rejected alternatives:**

- **Cross-field consistency checks** like "IRBinOp.op1 and IRBinOp.op2 widths match `width`". The widths of operands aren't stored on `IROperand` (they live in `IRBinOp.width` for both); the cross-check would have to happen at lowering time. Skipped — out of scope for k7al.

- **`@assert` instead of `error()`** for the constructor body. CLAUDE.md §1 ("FAIL FAST, FAIL LOUD") prefers explicit `error(...)` with context over `@assert`, and `@assert` is stripped by `--check-bounds=no`. Used `error()` everywhere; matches the existing pattern in the codebase.

- **`Union{Int, Nothing}` for IRSelect.width / IRPhi.width** instead of the `0` pointer sentinel. Cleaner Julia type-wise but: (a) breaks every existing call site that reads `.width` arithmetically, (b) the M2b sentinel convention is well-established with comments at the emitter sites, (c) lower.jl's `_pick_alloca_strategy` and friends already check `width == 0` for pointers. Kept the sentinel; documented it.
