## Session log — 2026-05-15 — Bennett-8ygo + Bennett-bipw close: soft_atanh + soft_acosh log1p refactors

**Shipped:** symmetric follow-ups to Bennett-tfmo (asinh) — `soft_atanh`
in `src/softfloat/fatanh.jl` and `soft_acosh` in `src/softfloat/facosh.jl`
both rewritten to Julia stdlib's natural log1p-based forms.

**atanh** (Bennett-8ygo, drops K=25 polynomial + 26 coefficients):
- (a) `|x| < 0.5`     → `sign(x) · 0.5·log1p(2|x|/(1-|x|))`
- (b) `0.5 ≤ |x| ≤ 1` → `sign(x) · 0.5·log((1+|x|)/(1-|x|))`
- `|x| > 1` → `NaN`

**acosh** (Bennett-bipw, drops K=15 polynomial + 16 coefficients + the
`s² = 2(x-1)` substitution scaffolding):
- (b) `1 ≤ x < 2`     → `log1p(t + sqrt(2t + t²))` where `t = x-1`
- (c) `2 ≤ x < 2^28`  → `log(2x - 1/(x + sqrt(x²-1)))`
- (d) `x ≥ 2^28`      → `log(x) + ln(2)`
- `x < 1` → `NaN` (domain)

**Why these were trivial after tfmo.** Each just substitutes the
matching Julia stdlib formula. atanh in particular gets a cleaner
implementation than asinh because there are only two in-domain
regimes (no huge arm needed — `|x| ≤ 1` is bounded). acosh keeps
the domain-restricted §13 contract: `soft_acosh(subnormal) = NaN`
because subnormals are < 1.

**Validation.** atanh: test_softfatanh.jl 2087/2087; LLVM dispatch
33/33; 0 ULP on most spot-checks, max 1 ULP at x=0.1. acosh:
test_softfacosh.jl 4492/4492; LLVM dispatch 27/27; max 1 ULP at
x=1.0001 and x=1.001.

**Gotchas / Lessons.**
* atanh's regime-a result for subnormal |x|: 2|x| stays subnormal,
  (1-|x|) rounds to exactly 1.0, ratio = 2|x|, `soft_log1p(subnormal) ≡
  subnormal` bit-exactly, halved = |x|. So the §13 contract holds via
  log1p's tiny-arg fast path WITHOUT an explicit `is_tiny` regime —
  the polynomial proof from Bennett-g82n was load-bearing, now log1p
  carries the same property by construction.
* acosh's regime-b at x=1 exactly: t=0, sqrt(2·0 + 0²)=0, log1p(0)=0
  bit-exactly. The old polynomial branch went through s²·kernel(s²)
  with the same s=0 short-circuit; the new path is one fewer level of
  computation.
* No medium/huge edge cases — both refactors validated through the
  existing test suites which already cover boundary inputs.

**Next agent starts here:** All three inverse hyperbolics now use
soft_log1p. The remaining Tier C2 transcendentals (Bennett-0ulc log1p
and Bennett-o7cy expm1) are already closed; the rest of the soft-float
library doesn't have similar polynomial workarounds.

## Session log — 2026-05-15 — Bennett-tfmo close: soft_asinh K=30 polynomial → soft_log1p (Julia-stdlib natural formula)

**Shipped:** `src/softfloat/fasinh.jl` rewritten — drops the 30 coefficient
constants + 60 Horner-Taylor ops in favour of Julia stdlib's natural
four-regime recipe:
- (a) `|x| < 2^-28` → return `a` (sign + subnormal preserved bit-exactly);
- (b) `2^-28 ≤ |x| < 2` → `sign(x)·log1p(|x| + x²/(1 + sqrt(1+x²)))`;
- (c) `2 ≤ |x| < 2^28` → `sign(x)·log(2|x| + 1/(sqrt(x²+1) + |x|))`;
- (d) `|x| ≥ 2^28` → `sign(x)·(log(|x|) + ln(2))`.

13 soft-float ops total (1 fmul + 4 fadd + 1 fsqrt + 2 fdiv + 1 log1p +
1 log + 3 fadd in the cascade), all unconditional — vs ~60 ops for the
Bennett-sfx9 K=30 polynomial.

**Why now:** worklog 062 explicitly filed this as future work when
Bennett-sfx9 landed asinh under the constraint "Bennett.jl has no
soft_log1p". Tier C2.1 (Bennett-0ulc) closed log1p two weeks later;
this is the natural pickup. Same opportunity exists for `soft_acosh`
and `soft_atanh` — separate follow-ups, not bundled here.

**Gotchas / Lessons.**
* Regime (c) uses the small-arg log trick `log(2|x| + 1/(sqrt(x²+1)+|x|))`
  not Bennett-sfx9's `log(|x| + sqrt(x²+1))`. The argument stays close
  to `~2|x|` at moderate |x|, sidestepping the soft_log precision loss
  that motivated the original polynomial workaround. Mathematically
  identical to asinh but numerically safer.
* The polynomial-regime subnormal-input proof from Bennett-sfx9
  (`x² → 0` → kernel returns 1.0 → fmul preserves x) is now GIVEN by
  regime (a)'s explicit `is_tiny` fast path — any subnormal |x| ≪ 2^-28,
  so the cascade returns `a` bit-exactly without computing anything.
* All 26 strategic spot-checks (zero/one/regime-boundary/Inf/NaN/huge)
  return **0 ULP** vs `Base.asinh` — even tighter than the ≤2 ULP
  budget. The Julia stdlib recipe is more accurate than the K=30 Taylor
  workaround.
* Non-finite handling matches `Base.asinh`'s `!isfinite(x) && return x`:
  ±Inf returns the input bits unchanged; NaN returns `a | QUIET_BIT`
  (forcibly quieted, matching the file's pre-tfmo behaviour).

**Rejected alternatives.** None considered — the Julia stdlib recipe
is the gold standard the original Bennett-sfx9 was explicitly
substituting for. Once `soft_log1p` exists, there's no reason to keep
the workaround.

**Validation.** `test_softfasinh.jl` 5178/5178 (including 300k random
× 3 seeds + subnormal-input §13 contract sweep + boundary tests);
`test_sfx9_llvm_asinh_dispatch.jl` 31/31 (compiles + runs through
both `llvm.asinh.f64` intrinsic and libm `@asinh` external-call
dispatch paths); standalone `soft_asinh` compiles + `verify_reversibility=true`
(13.2M gates / 7.98M wires under default strategies).

**Next agent starts here:** tfmo closed. Two siblings open as
follow-ups under the same opportunity: `soft_acosh` (Bennett-eq9p
work, K=15 polynomial via s²=2(x-1) substitution — could potentially
collapse to a log1p-based formula) and `soft_atanh` (Bennett-g82n,
K=25 polynomial in z=x² — atanh has the cleanest log1p form
`0.5·log1p(2x/(1-x))` for the medium regime). Neither filed as a
bead yet.

## Session log — 2026-05-15 — Bennett-9wmk attempt + revert: mid-algorithm fast_copy swap trades wires for depth

**Attempted:** Sun-Borissov Algorithm 3 §IV.B mid-algorithm `fast_copy`
swap in `lower_mul_qcla_tree!` — uncompute step 2 and step 1 BEFORE
`pa_tree`, `free!()` their slots so `pa_tree` reuses them via the
allocator's pop-from-free-list-first behaviour, then redo step 1+2 to
repopulate before uncomputing partial_products.

**Reverted.** Trade-off is not a clean win:
- wire_count: 2.43× → 1.77× paper across W ∈ {4,8,16,32} (~25% reduction)
- total gate count: −5% → +16% over paper (added 4(W²−W) re-emitted CNOTs at large W)
- total depth: +30% → +60% over paper at W=32 — **breaks the polylog-depth
  promise** that's `mul=:qcla_tree`'s entire value proposition vs
  `mul=:shift_add`.
- Toffoli count: unchanged ≈ paper.

**Why it failed the acceptance criterion.** Bead 9wmk's body claims
"implement the mid-algorithm swap to hit the paper's bound" (3n² ancilla),
but the test file's own comment notes the paper bound additionally
requires *`parallel_adder_tree` pool recycling*. The bare swap shaves
~25% (from ~2.43× to ~1.77×) — the rest needs pa_tree-internal recycling.
The 9wmk bead body and acceptance criterion are inconsistent on this.

**Gotchas / Lessons.**
* **Don't underestimate the depth metric.** The wire-granular depth
  walker in `test_mul_qcla_tree_paper_match.jl` counts a control wire's
  reads as causal too, so re-emitting 4(W²−W) CNOTs adds linearly to
  depth. The paper's Schedule-B time-slice model doesn't penalise this,
  but ours does.
* W=1 is fragile in this scheme: pa_tree's W=1 trivial branch allocates
  `final_out = allocate!(wa, 2)` immediately and pops the freshly-freed
  `y_bit_copies` slot for `final_out[1]`. The subsequent redo step then
  CNOT-overwrites the result bit. For W≥2 this didn't happen because
  pa_tree's internal pad/QCLA allocations exhaust the free list before
  `final_out` is reached. Empty-vcat-to-`Vector{Any}` was a separate
  gotcha at W=1 — use typed comprehension `Int[w for …]`.
* **`free!()` IS safe inside a self-cleaning sub-circuit under outer
  Bennett wrap.** I had been worried this triggered the Bennett-3of2
  failure mode, but `verify_reversibility=true` + exhaustive Int8 sweep
  passed (289/289 correct) even under the outer wrap. The 3of2 issue
  is specific to wire reuse where the reused wire's later "life" reads
  the **result** of the earlier life, not the case here where both
  lives independently round-trip a 0→value→0 cycle.

**Bead state.** Bennett-9wmk left OPEN with the empirical finding
captured in `bd notes`. A future implementer should bundle the swap
with `pa_tree` internal pool recycling so the depth blow-up is
avoided AND the ancilla hits the paper bound. Standalone swap isn't
worth landing.

## Session log — 2026-05-15 — Bennett-gsxe close: Cuccaro 2W−2 → 2W−3 Toffoli (§3.5 high-bit fold)

**Shipped:** `lower_add_cuccaro!` in `src/adder.jl` now applies the Cuccaro
2004 §3.5 high-bit optimisation. Test `test_op6a_cuccaro_gate_count.jl`
rebaselined to Toffoli = 2W−3, Total = 6W−5 (was 2W−2, 6W−4). Docstring
W=32 comparison numbers in `src/lowering/arith.jl` refreshed (410→408
total / 124→122 T-depth on `(x,y)->x+y` compile-wrapped).

**The trick.** At the W-1 boundary, the Toffoli inside the last MAJ
writes c_W into a[W-1] purely so that Phase-2's CNOT(a[W-1], b[W]) can
XOR c_W into b[W]; Phase-3's first UMA then immediately uncomputes c_W
back out of a[W-1] using the matching Toffoli. The §3.5 fold: drop both
Toffolis and inject ONE new Toffoli into Phase 2 with the same two
controls — (a[W-2], b[W-1]) for W≥3 or (X[1], b[1]) for W=2 — writing
into b[W] directly. Algebra: `(c⊕a)(b⊕a) = MAJ(a,b,c) ⊕ a`, and Phase-2
already XORs the spurious `a_{W-1}` term into b[W] via its second CNOT,
so the moved Toffoli + the existing CNOTs sum to exactly `b_W ⊕ a_W ⊕
c_W = s_W`. Wire a[W-1] (resp. a[1] at W=2) retains its original input
value across the boundary instead of transiently holding c_W. Net:
−1 Toffoli per call; −2 Toffoli in the Bennett-wrapped circuit (forward
+ reverse).

**Gotchas / Lessons.**
* W=2 needs a separate code path: the middle-MAJ loop is empty so there
  is no "last middle MAJ" to drop a Toffoli from — instead drop the
  FIRST MAJ's Toffoli, and balance with the Phase-3 *last* UMA's Toffoli.
  If you drop only one of those two, the last-UMA Toffoli reads
  (X[1]=a_1, b[1]=b_1⊕a_1) and corrupts a[1] back to a_1·b_1. The two
  drops must be matched.
* `test_gboa_dirty_bit_hygiene.jl` is the load-bearing correctness
  anchor: it sweeps (a_val, b_val) exhaustively for W ∈ {2,3,4,8} and
  pins {a preserved, b ← a+b mod 2^W, ancilla X = 0}. Survived the
  rewrite unchanged because the contract is implementation-opaque.
* Compile-wrapped saving is doubled (−2 not −1) because Bennett's
  forward + reverse copies the lowering twice. test_spa8's "Cuccaro
  T-depth ≥ 1.5× ripple" relative check is unaffected.

**Rejected alternatives.** None on the trick itself — the optimisation
is mechanical once the algebra is verified. 3+1 protocol not invoked:
`adder.jl` is not on CLAUDE.md §2's core-file list and the bead carries
a precise paper citation, a single scalar metric (−1 Toffoli per call),
and an exhaustive opaque correctness anchor (gboa).

**Validation.** Targeted tests: op6a 30/30, gboa 180/180, cuccaro_safety
893/893, gate_count_regression 39/39 (`:ripple` baselines untouched),
spa8 33/33, value_eager 1558/1558, add_dispatcher / xlsz / 5kio /
add_mul_cross / y986 all green. `BENNETT_HEAVY_TESTS=0 Pkg.test()` ran
clean post-edit.

**Next agent starts here:** gsxe closed. Adjacent algorithmic
optimisations in the same flavour are open as P3 beads: `q22p`
(Schedule B uncompute for parallel_adder_tree), `9wmk` (mid-algorithm
ancilla recycling via fast_copy swap), `h0ai` (auto self_reversing

## Session log — 2026-05-14 — Bennett-k31q close: NOT a test-ordering bug — it's `--check-bounds`

**Shipped:** `test/test_t0_preprocessing.jl` — capture the full extractor
error message in `skipped` instead of truncating at 80 chars; truncate
display-only (char-safe `first`); print the active `--check-bounds` mode;
header warning about check-bounds sensitivity. See git log around this
session's commit. `_handle_memset_arm` deliberately left untouched.

**Root cause (the real one).** `Bennett-k31q` was framed — and twice
mis-handled — as a "test-ordering dependency": `test_t0_preprocessing.jl`
passes standalone but fails in full `Pkg.test()`. It is **not** an ordering
dependency and shares **no** mutable global state. The driver is
`--check-bounds`. `Pkg.test()` hard-codes `--check-bounds=yes`; a bare
`julia --project test/...` uses the default. Under `--check-bounds=yes`,
`cond_pair` (`x -> [x,-x][1+Int(x<0)]`) can no longer stack-allocate its
array literal — Julia emits a GC frame, which brings a **volatile**
`llvm.memset`, an **inline-asm** thread-pointer read (`movq %fs:0`), and an
`ijl_gc_small_alloc` **external call**. All three are correctly fail-loud
rejected by Bennett (out of scope, not gaps). Under the default flag the
same function compiles to a plain `alloca [18 x i8]` — no memset, no GC
frame, extracts clean. Confirmed: the test fails standalone too the moment
you pass `--check-bounds=yes`, and passes under `Pkg.test()` after the fix.

**The actual bug.** `test_t0_preprocessing.jl` already has an allowlist for
exactly these benign fail-loud skips — it contains `"volatile"`,
`"inline-asm"`, `"call ptr asm"`, `"atomic store"`. But the catch block
captured `sprint(showerror, e)[1:min(80,end)]`, and Bennett's verbose
fail-loud messages put the diagnostic keyword well past char 80 (the
`cond_pair` message is 399 chars; `"volatile"` sits at ~char 130). So the
allowlist grep never saw the keyword and every long message looked
"unexpected". Fix: capture the **full** message; truncate display-only
(char-safe `first(msg, 160)`, so the multibyte `—` in Bennett error
messages never splits on a byte boundary).

**Rejected alternative.** Subagent A's recommended fix was to reorder the
`_handle_memset_arm` predicate cascade so the `c==0` silent-drop fires
before the volatile check. Rejected: (1) it would be a core-file change
(3+1) for no real benefit — `cond_pair` hits inline-asm two instructions
after the memset, so "fixing" the memset just moves the failure to the next
construct; (2) `_handle_memset_arm` is *correct* — a volatile memset
genuinely cannot be honoured, and any Julia function carrying a volatile
GC-frame memset also carries the inline-asm + GC alloc that come with a GC
frame, so it is un-compilable regardless. The bug was never in the extractor.

**Gotchas / Lessons.**
* `test_t0_preprocessing.jl` is `--check-bounds`-SENSITIVE. It MUST be
  validated under `--check-bounds=yes` (= the `Pkg.test()` configuration). A
  standalone `julia --project test/...` run uses the default and will not
  reproduce the check-bounds=yes-only skips — this is exactly what got k31q
  wrongly closed on 2026-05-13. The test now prints its active check-bounds
  mode and warns when it is not `=yes`.
* Truncating a captured error message *before* keyword-matching it is a trap:
  the truncation silently neuters the allowlist. Match on the full string;
  truncate only for human display, and char-safe.
* Two Sonnet subagents (general-purpose, parallel, blind) both independently
  identified `--check-bounds` as the root cause. Subagent A confirmed it with
  real `--check-bounds=yes` IR; subagent B's specific-mechanism guesses
  (ConstantExpr byte count / atomic memset variant) were wrong because it
  never ran the check-bounds=yes IR — but its root-cause *ranking* was right.
  Lesson: a subagent that runs the actual reproduction beats one reasoning
  from code alone — and the orchestrator must still verify (subagent A's
  *recommended fix* was wrong even though its *diagnosis* was right).

**Validation:** `test_t0_preprocessing.jl` 2/2 green under both
`--check-bounds=yes` (the previously-failing config) and the default; full
`Pkg.test()` re-run to confirm the suite is green (the prior 3 latent-red
failures — 2 kmuj stale counts + this one — are now all resolved).
