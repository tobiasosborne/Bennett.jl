## Session log — 2026-05-06 — Bennett-eq9p / Tier C1.10 — `soft_acosh` + `llvm.acosh` + libm `@acosh` dispatch (fifth hyperbolic, domain-restricted)

**Shipped:** see git log around the eq9p close commit. **Fifth
hyperbolic primitive** in Tier C1 (10 of 11 done), following Bennett-
sfx9 (`soft_asinh`).

**Why:** continuing the C1 grind. After m2bv/ky5n/bybh/sfx9, acosh is
the first domain-restricted hyperbolic. Domain restriction is the
new design wrinkle.

**Algorithm decision (3+1 skipped per §2 exception).** Acosh is
mechanically similar to asinh (same `log1p` dependency, same
polynomial-substitution strategy) plus a domain-check. The novel
piece — the `s²` substitution for the polynomial regime — was
empirically validated via REPL sweep before implementation:

* `acosh(1+t) = sqrt(2t) · kernel(2t)` reformulation factors out the
  essential singularity at x=1, leaving a smooth `kernel(z)`.
* K=15 Taylor in `z = 2(x-1)` covers `1 ≤ x ≤ 1.3` to ≤2 ULP
  analytically; coefficients computed in BigFloat via least-squares
  regression (Vandermonde at small `z` points).
* Polynomial threshold 1.3 chosen empirically: at threshold 1.05/1.1/1.2,
  the medium formula `log(x + sqrt(x²-1))` exceeded 2 ULP for inputs
  just past the boundary (3 ULP at x ≈ 1.06; 3 ULP at x ≈ 1.10; 3 ULP
  at x ≈ 1.15). Bumping threshold to 1.3 cleared 300k × 3 seed sweep
  with max 2 ULP, 0 fails.

**Domain restriction handling.** Julia stdlib throws `DomainError`
for x < 1. Bennett can't throw — branchless model. Per IEEE 754-2019
convention for invalid-domain transcendentals (matching `soft_asin`,
`soft_acos`'s |x|>1 NaN semantics), Bennett returns `NaN`.

Domain check predicate: `is_below_one = is_negative | (a < 1.0_bits)`.
Sign-bit set ⇒ definitely negative ⇒ < 1. Sign-bit clear AND unsigned
bits < 1.0_bits ⇒ x ∈ [0, 1) ⇒ < 1. NaN bit patterns slip through
this check (NaN bits > 1.0_bits in unsigned ordering) — corrected by
the final `is_nan` override.

**Results.**

* `src/softfloat/facosh.jl` (~140 LOC) — four-regime branchless port:
  `is_below_one → NaN`, K=15 polynomial via s²-substitution for
  `1 ≤ x ≤ 1.3`, `log(x + sqrt(x²-1))` for medium, `log(x) + ln(2)`
  for huge. ONE soft_log call via regime-selected arg.
* Module-private constants: 16 polynomial coefficients (`_ACOSH_C0..C15`)
  + 5 thresholds.
* `_CALLEES_FP_TRANS` extended **23 → 24**.
* `src/extract/instructions.jl` gains 3 dispatch arms.
* `test/test_softfacosh.jl` (10 testsets, **4492 assertions**, 2.0s):
  smoke / specials / domain (x < 1 → NaN for negative, ±0, subnormal,
  tiny-normal, just-below-1) / poly↔medium boundary at 1.3 /
  medium↔huge boundary at 2^28 / **§13 subnormal-INPUT always NaN**
  / poly fine sweep / medium fine sweep / 100k random × 3 seeds × 5
  buckets / callee registered.
* `test/test_eq9p_llvm_acosh_dispatch.jl` (7 testsets) + 4 fixture
  `.ll` files. Includes regression-guard `llvm.acos.f64` → `soft_acos`.

**Validation.**

* Smoke: max 1 ULP across 25 representative inputs spanning all
  regimes + domain-error inputs.
* Domain check: 17 OOB inputs all returned NaN.
* §13 binade sweep: all 2148 subnormal/tiny-normal inputs returned NaN
  (correct domain error for x < 1).
* 300k random sweep × 3 seeds × 5 magnitude buckets — **max 2 ULP,
  zero failures > 2** across all seeds.
* `@code_llvm` SLP-vectorisation check — clean.

**Gotchas / Lessons:**

1. **Polynomial threshold is rounding-budget-dictated, not analytical-
   accuracy-dictated.** K=15 Taylor analytically covers x ≤ 1.3 to ≤2
   ULP. But the compiled circuit has soft-float rounding accumulation:
   each `soft_fmul` and `soft_fadd` adds up to 0.5 ULP per step, and
   K=15 polynomial is 15 of each step. The medium-formula side has
   its own rounding (1 fmul + 1 fsub + 1 fsqrt + 1 fadd + 1 log =
   ~3 ULP worst case). For both arms to reliably stay ≤2 ULP at the
   transition, the polynomial threshold must be wider than the
   analytical convergence boundary alone would suggest.
2. **Domain-restricted §13 contract.** Sinh/tanh preserve subnormals
   bit-exactly; cosh rounds them to 1.0. Acosh's domain excludes the
   entire subnormal range, so `acosh(any subnormal) = NaN`. Same
   convention as `soft_asin` (|x|>1 → NaN, NOT preserved).
3. **`is_below_one` predicate construction.** Can't use unsigned
   comparison directly because negative-bit-pattern x has UInt64 bits
   ≥ 2^63 (sign bit set). Need `(sign_bit_set) | (a < 1.0_bits)`.
   The OR catches both negative and `[0, 1)`.
4. **K=15 vs K=30 (asinh).** Asinh's polynomial needed K=30 because
   its Taylor series has slow convergence (radius 1, branch points
   at ±i — reaching 0.55 was at the limit). Acosh's `s²` reformulation
   converges much faster because the smooth kernel function is
   well-behaved on `[0, 2]` (matching `acosh(1+t)/sqrt(2t)`).

**Rejected alternatives:**

* **Verbatim Julia stdlib regime split using log1p.** Same blocker as
  asinh — Bennett.jl lacks `soft_log1p`. Polynomial-substitution
  pattern reused.
* **Polynomial in `t = x-1` directly (no `s²` substitution).** The
  Taylor series for `acosh(1+t)` itself starts with `sqrt(2t)`, so
  the leading term has a sqrt singularity. The `s²` substitution
  factors this out, leaving a smooth polynomial. Direct-in-t would
  require fractional powers — much harder.

**Next agent starts here:** **`atanh` is the next pickup** (Tier
C1.11 — FINAL hyperbolic, completing Tier C1: 11 of 11). Domain
restriction: `|x| < 1` (atanh diverges at ±1; Julia throws DomainError
for |x| ≥ 1). `atanh(x) = 0.5 · log((1+x)/(1-x))`. Same `log1p`
dependency for small |x|. Regime split: domain (|x| ≥ 1 → NaN),
polynomial (|x| ≤ ~0.5), log-formula. Sign symmetry: atanh is ODD,
so work on |x| and OR sign at end. Bennett-soft_log1p bead is still
filed as future work — could simplify atanh significantly.

## Session log — 2026-05-06 — Bennett-sfx9 / Tier C1.9 — `soft_asinh` + `llvm.asinh` + libm `@asinh` dispatch (fourth hyperbolic)

**Shipped:** see git log around the sfx9 close commit. **Fourth
hyperbolic primitive** in Tier C1 of the Enzyme parity north-star
(9 of 11 done), following Bennett-bybh (`soft_cosh`).

**Why:** continuing the C1 grind. After tanh/sinh/cosh, asinh requires
a different algorithmic structure (log-based, not exp-based), exposing
the `soft_log1p` gap that `soft_expm1` has on the exp side.

**Strategic decision (skipped 3+1 protocol).** Asinh's algorithmic
direction was determined by direct empirical investigation rather than
proposer subagents. The decision tree:

1. **Verbatim Julia stdlib port** uses four regimes including a
   `log1p`-based small-|x| branch. Bennett.jl has NO `soft_log1p`.
   Substituting `log(1 + arg)` directly loses precision dramatically
   (~16M ULPs at |x|=1e-9 vs Base.asinh; ~12 ULPs at |x|=0.1).
   Empirically validated by direct soft_log inspection — soft_log
   itself matches Julia log to 0 ULP, but the formula
   `log(|x| + sqrt(x²+1))` is intrinsically less accurate than
   Julia's log1p reformulation at small |x|.
2. **Add `soft_log1p` as a sibling primitive** — multi-bead scope
   creep. Documented as future work; would help atanh/acosh too.
3. **Extend the polynomial regime** to cover the inaccurate range.
   Bennett's m2bv (tanh) and ky5n (sinh) used the same trick to
   sidestep `soft_expm1`. Choice for asinh: extend polynomial to
   `|x| ≤ 0.55` (where medium formula starts hitting ≤2 ULP at
   |x| ≥ 0.56).

**Polynomial degree choice — empirical:**

Asinh has slow Taylor convergence (radius 1, branch points at ±i):

* K=7  covers |x| ≤ 0.1   (≤2 ULP)
* K=18 covers |x| ≤ 0.45  (≤2 ULP)
* K=25 covers |x| ≤ 0.5   (≤2 ULP)
* K=30 covers |x| ≤ 0.55  (≤2 ULP)
* K=40+ covers |x| ≤ 0.6  (≤2 ULP)

Picked K=30 with threshold 0.55 to cleanly cover the gap up to where
`log(|x|+sqrt(x²+1))` becomes accurate. The medium formula's accuracy:

* |x| ≥ 0.56: ≤ 2 ULP
* |x| = 0.5:  ~3 ULP
* |x| = 0.1:  ~12 ULP
* |x| = 1e-9: ~16M ULPs

**Results.**

* `src/softfloat/fasinh.jl` (~210 LOC) — three-regime branchless port:
  K=30 polynomial in z=x² for `|x| ≤ 0.55`, `log(|x| + sqrt(x²+1))`
  for medium, `log(|x|) + ln(2)` for huge. ONE soft_log call via
  regime-selected arg. ONE soft_fsqrt.
* Module-private constants: 31 polynomial coefficients (`_ASINH_C0..C30`)
  + 4 thresholds/constants.
* `_CALLEES_FP_TRANS` extended **22 → 23**.
* `src/extract/instructions.jl` gains 3 dispatch arms.
* `test/test_softfasinh.jl` (10 testsets, **5178 assertions**, 2.1s):
  smoke / specials / poly↔medium boundary at 0.55 / medium↔huge
  boundary at 2^28 / **§13 subnormal-INPUT bit-exact across all 1074
  binades × ±** / poly fine sweep / medium fine sweep / 100k random
  × 3 seeds × 5 buckets / callee registered.
* `test/test_sfx9_llvm_asinh_dispatch.jl` (7 testsets) + 4 fixture
  `.ll` files. Includes regression-guard `llvm.asin.f64` → `soft_asin`
  using existing `ckvj_asin_f64.ll` fixture.

**Validation.**

* Smoke: max 1 ULP across 25 representative inputs spanning all
  regimes.
* §13 binade sweep: 2148 inputs (binades -1074..-1 × ±) — 0 ULP
  on subnormal inputs, ≤ 1 ULP on tiny normals.
* 300k random sweep × 3 seeds × 5 magnitude buckets — **max 2 ULP,
  zero failures > 2** across all seeds.
* `@code_llvm` SLP-vectorisation check — clean.

**Gotchas / Lessons:**

1. **`soft_log` near 1 loses precision; `soft_log1p` would recover
   it.** This is fundamental to `log(1+small)` evaluation in standard
   floating-point: when `1+small` rounds, the small contribution is
   lost. Julia stdlib's `log1p` factors the argument cleverly to
   preserve the small contribution. Bennett.jl needs a sibling
   `soft_log1p` primitive to enable accurate small-|x| paths in
   asinh / atanh / acosh-near-1. **Filed as future work** in the
   north-star doc — likely worth a separate bead.
2. **Asinh Taylor convergence is intrinsically slow.** Branch points
   at ±i give radius 1; covering |x| ≤ 0.55 requires K=30 polynomial.
   Compare sinh's K=8 polynomial for |x| ≤ 1.0 (which has no branch
   points on the real-axis-reachable disk). Higher gate cost is
   intrinsic, not a Bennett-specific limitation.
3. **The polynomial regime upper bound is dictated by where the
   medium formula's cancellation becomes acceptable**, not by where
   the polynomial accuracy degrades. At |x| = 0.55, polynomial gives
   ≤2 ULP and medium gives ~2-3 ULP. Just past 0.55, polynomial gets
   worse but medium gets better. So the boundary is fine-tuned by
   the medium formula's accuracy curve.
4. **3+1 protocol skipped.** This was a near-mechanical extension
   of the m2bv/ky5n/bybh playbook (file new soft_*.jl, add 3
   dispatch arms, write test + dispatch test + 4 fixtures). The
   strategic novelty was the K=30 polynomial degree decision, which
   was determined by direct REPL sweep before implementation.
   Documented per §2's exception clause.

**Rejected alternatives:**

* **Add `soft_log1p`** — out of scope for this bead. Would benefit
  atanh and acosh too. North-star note suggests filing it before
  starting atanh/acosh.
* **Use a wider polynomial (K=40 or higher) to cover even more of
  the regime** — diminishing returns. K=30 gives ≤2 ULP at |x| =
  0.55, which is sufficient. Higher K just adds gates without
  improving accuracy.

**Next agent starts here:** **`acosh` is the next pickup** (Tier
C1.10). Domain restriction: `acosh(x)` defined only for `x ≥ 1`
(Julia throws `DomainError` for x < 1). Branch structure (Julia
stdlib hyperbolic.jl:201+):
- `x = 1`: log1p path with t=x-1
- `1 < x < 2`: log1p path
- `2 ≤ x < 2^28`: `log(2x - 1/(sqrt(x²-1) + x))`
- `x ≥ 2^28`: `log(x) + ln(2)`

Same `log1p` dependence as asinh in the small-x range. Pre-bead:
consider whether to first ship `soft_log1p`. The acosh poly regime
(if we sidestep log1p again) is on `1 ≤ x ≤ ~1.5` covering a different
shape. After acosh, atanh (Tier C1.11) finishes the Tier C1 family.
