**Algorithm (Arm Optimized Routines / musl log.c, MIT-licensed port):**

  1. Special-case detection: NaN / ±Inf / ±0 / negative / subnormal flags
     extracted via integer bit-pattern ops (no `if` branches).
  2. Subnormal-input normalization: `_sf_normalize_to_bit52(fa, 1)` brings
     the leading 1 to bit 52 and decrements the effective biased exponent
     to a (possibly negative) Int64 — UInt64 wrap-around on the re-encoded
     `ix_eff` lets the `(int64)tmp >> 52` arithmetic-shift recover signed
     `k` correctly for subnormal inputs. Faithful reproduction of musl's
     `ix = asuint(x · 2^52); ix -= 52<<52` integer trick, but via the
     CLZ-based normalize helper rather than `soft_fmul`.
  3. Range reduction: `tmp = ix_eff - OFF` where `OFF = 0x3FE6000000000000`
     (≈ 0.6875). Index `i = (tmp >> 45) & 127` selects one of 128 table
     entries; `k = (Int64)tmp >> 52` recovers the signed exponent. `iz` is
     the re-encoded `z` ∈ [OFF, 2·OFF).
  4. Table lookup: `(invc, logc)` from a 128-entry table of UInt64 pairs
     hardcoded from `log_data.c`.
  5. `r = soft_fma(z, invc, -1.0)` — small (|r| < 1/(2N)).
  6. `kd = soft_sitofp(k)`; `w = kd·Ln2hi + logc`; `hi = w + r`;
     `lo = (w - hi) + r + kd·Ln2lo` — Cody-Waite split mitigates
     cancellation in the `k·ln2 + log(c)` sum.
  7. Polynomial: `y_main = lo + r²·A0 + r³·(A1 + r·A2 + r²·(A3 + r·A4)) + hi`,
     coefficients from Arm 2018 N=128, LOG_POLY_ORDER=6 (5 coeffs).
  8. **Near-1.0 path** (always computed, branchless-selected): degree-12
     polynomial in `rn = x - 1.0` with Dekker hi/lo split for the leading
     `rn + B0·rn²` term (B0 = -0.5 exact). Triggered when
     `(a - LO_BITS) < HI_BITS - LO_BITS` where `[LO, HI] = [1-1/16, 1+1.09/16]`
     — UInt64 unsigned compare automatically excludes negatives and
     out-of-range inputs. Without this path, `log(1.0)` returns ~1e-19
     (cancellation in `logc + log1p(r)`) and `log(1+1e-15)` is off by ~4·10¹¹
     ULP. With it, `log(1.0) == 0` bit-exact and the entire near-1 region
     hits ≤2 ULP.
  9. Special-case override chain (last-write-wins): zero → -Inf,
     negative → QNAN, +Inf → +Inf, NaN → propagate-with-quiet-bit.

**Accuracy result:** **100% bit-exact vs `Base.log` on 100k random samples**
across input range `[1e-300, 1e300]` (uniform in log-space). Better than
the ≤2 ULP target by a wide margin — surprising given `soft_exp` has ~0.9%
disagreement vs `Base.exp`. Likely Julia's `Base.log` (base/special/log.jl)
uses an algorithm in the same Arm/musl family rather than openlibm's
`e_log.c`. The empirical-100%-match contract is NOT pinned by a test —
only the ≤2 ULP contract is — so future Julia versions changing
`Base.log` won't trip the suite. Test sweep has the §13-mandated
`subnormal-INPUT range` testset (analogue of soft_exp's subnormal-OUTPUT
range; log's output is never subnormal so the §13 rule applies to the
input side).

**Gotchas / Lessons:**

- **Catastrophic cancellation near x=1 is fundamental to the main path,
  NOT optional.** First-pass implementation skipped Arm's near-1 path,
  hoping the main path's degree-6 polynomial would suffice. It didn't:
  ULP error grew to 4·10¹¹ at x=1+1e-15 because `k·ln2 + logc + log1p(r)`
  is `0 + small + small` with magnitudes that nearly cancel. The
  near-1 path uses a degree-12 polynomial in `rn = x - 1` *directly* —
  no table, no `log(c)` — so cancellation never occurs. Bennett's
  branchless style means computing both paths always; ifelse-select at
  end. The cost is acceptable for correctness.
- **`(a - LO_BITS) < (HI_BITS - LO_BITS)` is the standard "is in range"
  unsigned trick.** For negative x, sign bit is set so `a` is huge as
  UInt64 → `a - LO` is even huger → comparison fails. Cleanly excludes
  negatives without an extra check. For x in [LO, HI), the subtraction
  doesn't wrap. For x outside, it wraps to a value > range. Worth
  remembering for future unsigned-compare-as-flag patterns.
- **`_sf_normalize_to_bit52` returns Int64 exponent that can go negative
  for subnormals.** Casting back to UInt64 via `reinterpret(UInt64, ::Int64)`
  preserves the bit pattern, and `(ea_eff << 52)` then exhibits the
  expected wrap-around behavior. The `(Int64)tmp >> 52` arithmetic-shift
  recovers signed `k`. Tested with the §13 subnormal-input sweep.
- **Test bug: `check_log(ℯ; tol=...)`** failed because `ℯ` is `Irrational`,
  not `Float64`, and the typed `check_log(a::Float64;...)` doesn't auto-
  convert. Fixed by `check_log(Float64(ℯ); ...)`. Mention here as a
  reminder for future transcendental tests — `π`, `ℯ`, etc. are
  `Irrational`, not Float64.
- **Bennett's "branchless integer arithmetic" claim is about bit-pattern
  manipulation, NOT about bypassing soft_f*.** Same as `soft_exp`:
  polynomial body uses `soft_fmul`/`soft_fadd`/`soft_fma`/`soft_fsub`
  primitives, integer ops only for sign/exp/frac extraction and
  special-case dispatch. Each `soft_*` call is itself thousands of
  reversible gates, so a `soft_log` call will be ~50–100k gates.

**Rejected alternatives:**

- **Embedded log helper inside fpow.jl.** Closes emv but leaves 582 open,
  doubles internal vs public surface confusion (musl/Arm/fdlibm all keep
  pow's internal log_inline separate from user-facing `log`, but they
  also have different precision targets). Bennett's test discipline (one
  test file per primitive, §13 subnormal sweep per primitive) makes the
  separate-file path cleaner.
- **fdlibm-style algorithm (e_log.c, Sun Microsystems).** Tableless,
  ~1 ULP, uses `s = f/(2+f)` and odd-only series. Simpler code but
  harder to extend for pow's extended-precision needs. Arm's table-
  based path matches our existing `soft_exp` precedent and is the
  proven pattern.
- **Wider near-1 trigger range to absorb more inputs.** Arm's
  `[1 - 1/16, 1 + 1.09/16]` (≈ [0.9375, 1.068]) is calibrated to the
  cancellation region; widening it would slow the common case without
  improving accuracy outside the cancellation region.

**Empirical accuracy:**
- `soft_log` vs `Base.log`: 100% bit-exact across 100k random samples.
- `soft_log2` vs `Base.log2`: ≤1 ULP (often 0) on a 5k random sweep.
  Spot-checks: `log2(1)`, `log2(2)`, `log2(4)`, `log2(0.5)`, `log2(0.25)`,
  `log2(5)`, `log2(1e±100)` all bit-exact; `log2(8) = 2.999999999999...`
  (1 ULP); `log2(3)` and `log2(1e±300)` 1 ULP.
- `soft_log10` vs `Base.log10`: ≤1 ULP on a 5k random sweep. Spot-checks:
  `log10(1)`, `log10(10)`, `log10(100)`, `log10(1000)`, `log10(7)`,
  `log10(1e±50)`, `log10(1e±100)` bit-exact; `log10(0.1) = -0.999...`
  (1 ULP, the 1/0.1 round-trip irrationality bites).
