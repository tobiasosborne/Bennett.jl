**Gotchas / Lessons:**

- **`tail` filter on `Pkg.test` output is dangerous.** First Pkg.test run for 348q used `julia ... | tail -120`; output buffering through the pipe meant the captured output was a *15-line stack-trace fragment* despite exit code 0. The stack trace looked alarming but was actually unrelated noise — `Pkg.test` succeeded. Re-ran with `grep -E "tests passed|tests failed|Test Failed|^ERROR"` to filter for actual signal; got 30 lines ending in `Testing Bennett tests passed`. **Pattern for future**: when capturing background `Pkg.test`, filter by signal terms (`tests passed`/`Test Failed`), not by line-count tail.

- **Tuple return from a multi-method function is type-stable enough.** `_gate_controls` now returns `Tuple{}`, `Tuple{Int}`, or `Tuple{Int,Int}` depending on gate type. Inferred return type at call sites is the union, but Julia's union-splitting handles iteration without a runtime dispatch cost — same code generation as the prior `Vector{Int}` path, minus the heap allocation. No measurement degradation in any of the 1293 assertions in the `_gate_controls`-touching test suites I ran standalone (`test_mlny_depth.jl`, `test_toffoli_depth.jl`, `test_pebbling.jl`, `test_eager_bennett.jl`).

- **Dispatch dicts must be defined AFTER the dispatched-to functions exist.** `_lower_load_via_mux_2x8!` and friends are defined by `@eval` at the bottom of `lower.jl` (lines ~2640-2728). The new `_MUX_EXCH_LOAD_DISPATCH` const must therefore live AFTER that `@eval` block — which I placed it. Functions used INSIDE the dict are referenced by name and resolved at module-init time when the const is constructed, NOT at file-parse time. Future agents adding similar dispatch tables: place them after every function they reference, not before.

- **Bead-tfo8's claim of "shape set hardcoded twice" undercounted.** Actual count is FOUR places: `_pick_alloca_strategy` (lists all 6 shapes), the load dispatcher (6 elseif arms), the store dispatcher (6 elseif arms), and the `@eval` generator (lists 4 shapes — only the auto-generated subset). Collapsed the first three; the `@eval` shape list stays as-is because lm3x is the bead for unifying hand-written vs @eval-generated bodies. Caught the actual count via `grep -n "_lower_load_via_mux\|_lower_store_via_mux"` — counted occurrences before editing.

- **The new tfo8 invariant test caught its own design issue early.** First version of `test_tfo8_alloca_strategy_tables.jl` constructed `IROperand(:const, :unused, UInt64(0), 0)` — wrong: the struct has 3 fields and `value::Int` not `UInt64`. The test errored on the type signature mismatch before any assertion ran. Switched to the existing `Bennett.iconst(0)` and `Bennett.ssa(:idx)` helpers from `ir_types.jl:9-10`. Pattern: prefer existing constructor helpers over re-creating struct positional args.

**Rejected alternatives:**

- **Including the `@eval` block's shape list in `_MUX_EXCH_STRATEGY`** as a single source. Would have meant emitting the hand-written (4,8)/(8,8) helpers via `@eval` too, which is exactly what `Bennett-lm3x / U56` is about — and lm3x is gated on `Bennett-qcse / U51` export stabilisation. Out of scope for tfo8. Crisp boundary: tfo8 unifies the *dispatch-time shape set*; lm3x unifies the *generation-time shape set*.

- **Eliminating `:shadow` and `:shadow_checkpoint` branches into the dispatch dict.** Tempting for symmetry. Rejected because they require *different* function signatures (`_lower_store_via_shadow!` takes `info` AND `block_label`; `_lower_store_via_shadow_checkpoint!` takes both; the MUX EXCH variants take neither in the load case and only `block_label` as kwarg in the store case). Forcing a uniform signature would have required wrapper closures. Cleaner to keep two explicit branches.

- **Replacing `Vector{Int}` in `eager.jl:41`'s upstream `last_use::Dict{Int, Int}` accumulator** as part of 348q. Out of scope — eager.jl uses `last_use[c]` indexing which expects scalar wire IDs, exactly the value the tuple iteration produces. The tuple change is fully local to `_gate_controls`'s return shape; downstream Dict mutation is unchanged.

- **Adding `sizehint!` opportunism (Bennett-5kio / U109)** while editing dep_dag.jl. Avoided scope creep — 5kio is a separate bead with its own analysis (142+69 push! sites) and would have meant another ~45-min audit of every loop. Future agents: if you're already in a hot file for another reason, it's tempting to bundle 5kio sites — resist unless the file is small.
