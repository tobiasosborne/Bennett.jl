**Gotchas / Lessons:**

- **Bare `catch` swallows `InterruptException`.** Standard Julia idiom: `catch e; e isa InterruptException && rethrow(); ...`. I knew the rule but seeing 11 violations in one file emphasises that the pattern was simply never adopted here. The static-inspection test (`test_uinn_catch_narrowing.jl`) is the institutional fix — a future bare catch added to `ir_extract.jl` is now a test failure, not an undiscovered behaviour bug.

- **The bead's site count was approximate.** Bead said 9 bare catches at specific (stale) line numbers; actual count was 11 when I `grep`-ed for `catch` in the current file. Two of the lines the bead cited were already typed `catch e` blocks (lines 989, 2631) that handled exception types explicitly. I still added the `e isa InterruptException && rethrow()` guard to those two for *uniform pattern* — a reader scanning for InterruptException safety can now see the property at every catch site, instead of having to mentally trace the typed-catch's fallback `rethrow()` path. Cost is one duplicate line per site.

- **Static-inspection tests are great for "you must adopt this idiom" invariants.** The uinn test is ~50 LOC of regex/string scanning; it has zero runtime cost (parses the source once at test start), zero false negatives in practice, and one false-positive class (a comment containing the word "catch" outside an inline-comment strip). Worth the effort because the alternative — testing that `InterruptException` actually propagates through a Ctrl-C during compilation — is fragile and hard to write deterministically.

- **First version of the static-scan test errored on `something(findfirst('#', line), nothing)`.** Julia's `something` requires at least one non-`nothing` arg. Replaced with explicit `hash_idx = findfirst('#', line); code = hash_idx === nothing ? line : line[1:hash_idx-1]`. Future agents writing string-scanning tests: explicit `=== nothing` checks beat clever `something()` chains.

- **Citations are diagnostic.** uzic was 25 minutes of work and adds zero behavioural value. But: (a) the bead was pre-existing review feedback, (b) the codebase is named after Bennett 1973 and didn't cite it, and (c) future agents reading `_add_at_most_k!` now see *Sinz 2005* spelled out instead of having to know the technique by heart.
