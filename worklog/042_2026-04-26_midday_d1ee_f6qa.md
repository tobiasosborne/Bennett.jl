**Gotchas / Lessons:**

- **`git log --grep` is the fastest way to find an "implementing commit" for a doc-status banner.** For 5ttt I needed each doc's SHIPPED-on commit. `git log --all --oneline --grep="Bennett-by8j\|memssa\|MemSSAInfo"` found `5411d1a` for memssa_investigation; the same pattern with `shadow|2ayo|oy9e` found `e16cf22` for shadow_design. Faster than reading `git log` chronologically. *Pattern*: when adding a "shipped via commit X" note anywhere, grep for the bead-id + topic, copy the first matching SHA.

- **`BENNETT_CI=1` is the right shape for a local fail-loud env var even on a CI-rejecting project.** CLAUDE.md §14 explicitly forbids GitHub Actions CI but doesn't prohibit local fail-loud modes. The pattern: default = silent skip on missing toolchain; opt-in = hard fail. A future contributor who DOES want CI-style strictness (e.g. before pushing a release) can `BENNETT_CI=1 julia ... -e 'using Pkg; Pkg.test()'` to surface every skip as a failure. Doesn't violate §14 because no GitHub workflow / scheduled run is involved.

- **Static-inspection tests are the third reuse of the pattern this week.** uinn (catch narrowing), f6qa (error message prefixes), now srsy (CI guards). All three pin a "this idiom must be present everywhere it applies" property by parsing the source file. Total ~50 lines of test code per such invariant; no runtime cost; catches future regressions reliably. *The pattern*: read the file, scan for the relevant markers, assert presence/absence.

- **Doc-only bead closes don't need Pkg.test.** 5ttt was strictly markdown edits in `docs/memory/`. The full Pkg.test takes ~5 min cold. Skipped it; just verified `using Bennett` still loads (it does — markdown isn't included by Bennett.jl).

- **`@test get(ENV, "BENNETT_CI", "0") != "1" || @info(...)` is a no-op-on-success Julia idiom.** Used in the srsy test to log when the env var IS set without failing. The `@info` returns `nothing`; Julia's `||` returns the last evaluated value. So either `BENNETT_CI != "1"` (test passes via short-circuit) or it equals "1" and `@info` runs as a side effect. Common Julia pattern; worth recording for similar conditional-logging tests.
