**Gotchas / Lessons:**

- **`grep -E 'error\("[A-Z][a-z]'` was the right invariant scan.** All 18 unprefixed sites started with a capital letter (`Unhandled`, `Unknown`, `Loop`, `GEP`, `VarGEP`, `Cannot`, `Phi`, `Block`, `No`, `Insufficient`, `Unmapped`, `Load`, `Undefined`). The grep flagged them all in one pass. *Future "fix all bare-prefix errors" beads in this codebase*: this regex is the standing test.

- **The static-inspection test for f6qa needs an _ALLOWED_PREFIXES list, NOT an offender pattern.** Tried both; the allowed-list form is more robust against future additions. Adding `_my_new_helper!:` as a prefix means appending to the const, not editing a regex. Future similar tests: prefer allow-listing.

- **Bead claims drift over time — third stale claim today.** d1ee said "QCLA depth formula is unasserted" → already pinned in test_qcla.jl:163-180. f6qa said "reversibility-check missing wire indices" → already fixed by Bennett-6azb / U58. f6qa said "Insufficient pebbles" wording inconsistent → was true (one had "at least", the other didn't). Triage pattern: read the cited test/source FIRST; close-as-already-fixed for stale halves; do the work for current halves; pin the historical claim via a static test so the bead can never reopen as stale.

- **Updating literal-message comments in test files matters.** test_t5_corpus_julia.jl had THREE comments quoting "GEP base thread_ptr not found in variable wires" (now `lower_var_gep!: GEP base ...`). Updated all three. *Pattern*: when changing an error message that's referenced verbatim in any comment / docstring, grep the repo for the literal string and refresh.

- **The `(supported: $_IR_BINOP_OPS)` etc. inline alphabet trick is reusable.** Three of the f6qa-edited messages now end with `(supported: $_IR_BINOP_OPS)` / `_IR_ICMP_PREDS` / `_IR_CAST_OPS`. The const tables (added in Bennett-k7al / U99 last session) interpolate cleanly into error messages. Future enum-typed errors: do the same.

- **The HAMT `@assert is_occupied + is_new == 1` only fires under research-tier tests** (BENNETT_RESEARCH_TESTS=1). The default Pkg.test path doesn't load HAMT, so a violation in default builds wouldn't surface. Acceptable because HAMT is research-tier; the assert is value when re-running with research enabled.

Settled on `<funcname-or-helper>:` (no file prefix), which matches lower.jl's pre-existing dominant pattern and is enough to attribute the message.
