`soft_atan2` is a faithful branchless port of musl `atan2.c` (FreeBSD/
SunPro 1993, BSD-licensed). **Built on `soft_atan`** (Bennett-qpke) per
CLAUDE.md ¬ß12: ONE `soft_fdiv(|y|, |x|)` + ONE `soft_atan(ratio)` call;
the rest is XOR / ifelse / one `soft_fsub` for the quadrant offset.
**Two LLVM-ingest paths** land in this bead ‚Äî the unary intrinsic
family had only one shape per function (`llvm.f.f64`), but atan2 ships
both:

1. `llvm.atan2.f64` intrinsic (LLVM 18+; this project bundles 18.1.7).
2. libm `@atan2(double, double)` external call (LLVM ‚â§17, `-O0` C/Rust,
   `-fno-builtin-atan2`, etc.).

`_CALLEES_FP_TRANS` extended 18 ‚Üí 19. f32 forms (both `llvm.atan2.f32`
and libm `@atan2f`) rejected per CLAUDE.md ¬ß13.

**Algorithm:** musl `atan2.c` reduces to `atan(|y|/|x|) + quadrant_offset`.
The four quadrant offsets:

- q0 (x‚â•0, y‚â•0): `z`             (no offset)
- q1 (x‚â•0, y<0): `-z`            (XOR sign bit)
- q2 (x<0,  y‚â•0): `œÄ - z`        (one fsub)
- q3 (x<0,  y<0): `z - œÄ`        (XOR sign bit on `œÄ - z`)

All four computed branchlessly; ifelse-cascade selects on `(sign_x,
sign_y)`. Special-case overrides (last-write-wins): `both_inf` ‚Üí
`¬±œÄ/4` or `¬±3œÄ/4`; `both_zero` ‚Üí `¬±0` or `¬±œÄ`; `is_nan` ‚Üí propagate
first-NaN-operand with quiet-bit set (matches Julia's `Base.atan` and
the x86 SSE convention).

The generic path handles every non-NaN edge naturally:
- `y=¬±0, x‚âÝ0 finite`: `0/x = +0; atan(+0) = +0`; offset gives ¬±0/¬±œÄ.
- `x=¬±0, y‚âÝ0 finite`: `y/0 = +Inf; atan(+Inf) = +œÄ/2`; offset gives
  ¬±œÄ/2 in all four quadrants (since `œÄ - œÄ/2 = œÄ/2`).
- `x=¬±Inf, y finite`: `y/Inf = +0; atan(+0) = +0`; offset gives ¬±0/¬±œÄ.
- `y=¬±Inf, x finite`: `Inf/x = +Inf; atan(+Inf) = +œÄ/2`; offset gives
  ¬±œÄ/2.

Only `0/0` and `Inf/Inf` produce NaN through the generic path; both
have explicit overrides.

**Investigation findings (the void-type rabbit hole):** before filing
the bead I tried `reversible_compile((y,x) -> atan(y,x), Float64,
Float64)` to see what failed. It threw `VoidType reached _type_width`
at `module_walk.jl:72`. Chasing this revealed two orthogonal facts:

1. Julia's `Base.atan(y, x)` does NOT lower to `llvm.atan2.f64` NOR to
   a libm `@atan2` external call. It inlines into LLVM IR using
   `fcmp / select / branch / llvm.fabs / llvm.copysign + a recursive
   `j_atan_*` call to the 1-arg form. So the LLVM-ingest path is only
   exercised by raw `.ll` / `.bc` (clang, rustc, etc.) ‚Äî not by Julia
   source. This is consistent with how the prior C1 closes shipped
   (qpke/ckvj/bd7f all tested only the `.ll`-ingest path).
2. The Julia-source path for ALL the recent transcendentals
   (sin/cos/tan/atan/asin/acos) is broken because there is NO
   `Base.<func>(::SoftFloat[, ::SoftFloat])` override in
   `softfloat_dispatch.jl`. Compare `Base.exp(x::SoftFloat) =
   SoftFloat(soft_exp_julia(x.bits))` (line 40, works) ‚Äî there is no
   analogous `Base.atan(x::SoftFloat)` etc. So the SoftFloat dispatch
   chain falls through Julia's generic `Base.atan`, which hits a
   void-returning code path.

This is a systemic gap in the Tier C1 family, not an atan2-specific
issue. Out of scope for this bead. **TODO**: file a follow-up bead for
"`Base.{sin,cos,tan,atan,atan2,asin,acos}(::SoftFloat[, ::SoftFloat])`
overrides" so `reversible_compile(<trig>, Float64[, Float64])` works
end-to-end via the SoftFloat dispatch path.

**Gotchas / Lessons:**

1. **Silent-miscompile root cause: `startswith(cname, "llvm.atan")` is
   too loose.** Pre-7goc, this prefix matched `"llvm.atan2.f64"` and
   dispatched to `soft_atan` with just `ops[1]` (the y operand),
   silently dropping x. Empirical probe with a `.ll` fixture containing
   `call double @llvm.atan2.f64(double %y, double %x)`: Bennett
   accepted the IR, emitted ~3.8M gates, simulated cleanly ‚Äî but
   produced wrong results in 5 of 8 quadrants (`atan2(3, 4)` returned
   `atan(3) ‚âà 1.249` instead of the correct `0.6435`; `atan2(0, -1)`
   returned `0` instead of `œÄ`; etc.). Per CLAUDE.md ¬ß1, this is a
   fail-loud violation. The fix that made the new arm work AT ALL ‚Äî
   adding `if startswith(cname, "llvm.atan2.")` BEFORE the existing
   `llvm.atan` arm ‚Äî also tightens all six trig prefixes
   (`llvm.{sin,cos,tan,atan,asin,acos}`) to require a trailing `.`,
   pre-empting the same class of bug for any future `llvm.{sinh,cosh,
   tanh,asinh,acosh,atanh}.f64` arm. **Lesson: when a dispatch arm uses
   `startswith` on an LLVM prefix that has natural sibling names
   differing by suffix character (atan/atan2/atanh), the `.` separator
   in the LLVM intrinsic naming convention IS the boundary ‚Äî encode
   it in the predicate.**

2. **Three "expensive primitive" calls collapse to two.** First-pass
   draft computed `pi_minus_z = soft_fsub(PI, z)` AND `z_minus_pi =
   soft_fsub(z, PI)` as two parallel fsubs. Same shape, no data
   dependency ‚Äî exactly the qpke gotcha #1 SLP-vectorisation hazard.
   Fix: `z_minus_pi = pi_minus_z ‚äª SIGN_BIT` (XOR-negation; correct
   for any finite `pi_minus_z`, and `z` is bounded by `œÄ/2 < œÄ` so
   `pi_minus_z` is never exactly `+0`/`-0` ambiguous). Same XOR trick
   for `neg_z = z ‚äª SIGN_BIT`. Final cost: ONE `soft_fdiv` + ONE
   `soft_atan` + ONE `soft_fsub`; everything else is bit ops.
   **Lesson: pairwise `soft_fneg` / `soft_fsub-from-constant` are
   replaceable by SIGN_BIT XOR whenever the operand is known finite,
   saving both gates and an SLP-vectorisation hazard.**

3. **NaN propagation matches Julia's `Base.atan(y, x)` first-arg-wins
   convention.** Reviewed `softfloat_common.jl::_propagate_nan2`: x86
   SSE rule is "if y NaN return y else x". Julia's `Base.atan(NaN,
   1.0)` returns NaN; `Base.atan(1.0, NaN)` also returns NaN. Both
   propagate ‚Äî the question is *which* payload survives.
   `nan_payload = ifelse(y_nan, y, x) | QUIET_BIT` matches both Julia
   and SSE. Confirmed via Pkg.test before close.

4. **The `both_zero` and `both_inf` overrides MUST come after the
   generic path.** First draft put `both_zero` first, but then a NaN
   input where `abs_y == 0 == abs_x` (impossible ‚Äî NaN's abs is NaN,
   not 0) wouldn't have been a problem. Belt-and-braces: cascade order
   is `generic ‚Üí both_zero ‚Üí both_inf ‚Üí is_nan`. Each later override
   strictly subsumes the wrong cases the earlier ones produce; NaN
   wins last because it's the most specific input class. Verified by
   exhaustive testset across all special inputs.

5. **Float64 constants `Float64(œÄ/4)`, `Float64(3œÄ/4)` round to
   exactly the bit patterns Julia and musl agree on.** Spot-checked
   `reinterpret(UInt64, Float64(3œÄ/4))` = `0x4002D97C7F3321D2`,
   matching musl `s_atan2.c`'s `pi_o_4` and `3*pi_o_4` constants
   bit-for-bit. No precision drift from the C-to-Julia transliteration.

**Rejected alternatives:**

- **Port `_atan2_with_pi_lo` precision correction (musl's `pi_lo`
  round-off compensation for the q2/q3 case)**: ~10 extra ops, gains
  one ULP on the q2/q3 boundary near `|y/x| ‚âà 1`. Bennett's contract
  is ‚â§2 ULP and the random sweep already meets it without the
  correction. Skipped to keep the implementation thin; can be added
  later if a benchmark flags the q2/q3 ULP as a problem.
- **Special-case the `atan(0/x) = 0` and `atan(y/0) = œÄ/2` paths to
  skip soft_fdiv when an operand is exactly 0**: avoids a wasted fdiv
  on a degenerate input. soft_fdiv handles ¬±0 and ¬±Inf operands
  cleanly already, so the branchless cascade gets the right answer
  without the special-case. Skipped ‚Äî fewer branches, simpler code.
- **Implement via `Base.atan(::SoftFloat, ::SoftFloat)` override
  instead of an LLVM-ingest dispatch arm**: would unblock the Julia-
  source path (`reversible_compile(atan, Float64, Float64)`) but
  doesn't help raw `.ll`/`.bc` ingest from C/Rust. The whole Tier C1
  family is missing the `Base.<f>(::SoftFloat)` overrides; a one-off
  for atan2 would be inconsistent. Tracked as a separate follow-up.
- **Three-way override fold (NaN ‚à® both_zero ‚à® both_inf in one ifelse
  cascade)**: tried; the readability loss outweighed the ~3 saved
  gates. Kept the explicit per-class overrides.
